# Setup Guide

Everything you need to browse, rebuild, or deploy the AI Exposure visualization.

---

## Quick start (just browse the visualization)

No Python needed. The dataset is already built and committed.

```bash
cd site
python -m http.server 8080
```

Then open [http://localhost:8080](http://localhost:8080).

> **Note:** You must use a local server — double-clicking `index.html` won't work because the app fetches `data.json` over HTTP.

---

## One-click start scripts

### Mac / Linux

```bash
./start.sh
```

Starts the server and opens your browser automatically.

### Windows

Double-click `start.bat`, or run it from a terminal.

---

## Full setup (rebuild or re-score data)

### 1. Install uv (Python package manager)

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Or see [docs.astral.sh/uv](https://docs.astral.sh/uv) for other install methods.

### 2. Install Python dependencies

```bash
uv sync
```

### 3. Add your OpenRouter API key

```bash
cp .env.example .env
# Edit .env and paste your key from openrouter.ai
```

### 4. Rebuild the site dataset

Reads `occupations.csv` + `scores.json`, writes `site/data.json`:

```bash
uv run python build_site_data.py
```

### 5. Re-score all occupations (optional)

Calls the OpenRouter API (Gemini Flash) for all 342 occupations. Takes ~10 minutes and costs roughly $0.50:

```bash
uv run python score.py
```

After scoring, run `build_site_data.py` again to rebuild `site/data.json`.

## Deploy to Vercel

1. Fork this repo on GitHub.
2. Go to [vercel.com](https://vercel.com) → New Project → import your fork.
3. Vercel auto-detects `vercel.json` — no manual config needed.
4. Every push to `main` triggers a fresh deploy.

---

## GitHub Actions (weekly refresh)

Workflow file: `.github/workflows/refresh.yml`

- Runs automatically every **Sunday at 06:00 UTC**.
- Rebuilds `site/data.json` with a fresh timestamp and auto-commits the result.
- Can also be triggered manually from the **Actions** tab in GitHub (workflow_dispatch).
- No secrets required for basic refresh.
- To also re-score occupations: add `OPENROUTER_API_KEY` under repo **Settings → Secrets → Actions**.

## Data sources

- **BLS Occupational Outlook Handbook** — scraped from bls.gov (342 occupations).
- **AI exposure scores** — generated by Gemini Flash via [OpenRouter](https://openrouter.ai), following [Andrej Karpathy's methodology](https://karpathy.ai) for estimating automation exposure.
